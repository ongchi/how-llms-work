How LLMs Work

From autocomplete to ChatGPT — in plain JavaScript

01 — Introduction

LLMs are Autocomplete at Scale

  • An LLM does one thing: predict the next token
  • "Given everything before, what word comes next?"
  • The wrapper (ChatGPT, Claude) adds system prompt, chat history, tool calls
Even a character-level frequency table 'learns' from data. Scale to billions of parameters + trillions of words = GPT-4.

03 — Embeddings

Meaning as Numbers

Each token is mapped to a point in a 50-dimensional vector space — words with similar meanings end up nearby.

  • Method: PPMI co-occurrence + SVD on Shakespeare
  • Count which words appear near each other
  • Weight by statistical surprise (PPMI)
  • Compress 1000D → 50D via truncated SVD
  • No human labels. Pure statistics.
The model discovers that "king" and "queen" are related — nobody told it.

04 — Attention

Context Changes Meaning

Static embeddings give every word one fixed vector. Attention lets each token look at all others and update its representation based on context.

Same word. Same initial embedding. Completely different contextualised meaning. GPT-4 has 96 attention heads doing this at every layer, for every token.
04 — Code 
// Load real 50D Shakespeare embeddings (prebuilt by scripts/build-embeddings.mjs)
const embeddings = await fetch("data/embeddings.json").then(r => r.json());

// Dot-product attention: each token scores all others, softmax → weights
function attend(sentence) {
  const words = sentence.split(" ");
  const vecs  = words.map(w => embeddings[w]);
  return words.map((word, i) => {
    const weights = softmax(vecs.map(v => dotProduct(vecs[i], v)));
    return { word, top: topWeights(words, weights, i) };
  });
}

// [{to:"queen",weight:0.154},{to:"his",weight:0.155},...]

// [{to:"battle",weight:0.163},{to:"and",weight:0.142},...]
05 — The Transformer Loop

Putting It Together

1 Tokenize "to be or not to..." → [51, 230, 36, 47, ...] §2
2 Embed IDs → 50-dimensional vectors §3
3 Attend ×N each token listens to all others §4
4 Logits linear projection → raw score per vocab word
5 Softmax(T) scores → probability distribution
6 Sample pick next token → go to step 1 → repeat
Temperature T: low = peaked distribution (predictable) · high = flat (creative)

06 — Training

How Models Learn

  • Start with random weights → make terrible predictions
  • Measure error → compute gradient → nudge weights → repeat
  • Self-supervised: every sentence on the internet is a training example — no labels needed
  • RLHF: human raters score outputs → reward model → fine-tune to be helpful, harmless, honest
~1Ttokens for GPT-3
175Bparameters (GPT-3)
07 — Beyond Text

CNNs, Multimodal & Diffusion

  • CNNs: slide small filter kernels across pixels — detect edges → shapes → faces
  • Multimodal (GPT-4o, Gemini, Claude): CNN vision encoder + language model
  • Diffusion (Stable Diffusion, DALL-E): start with pure random noise, iteratively denoise → sharp image
A CNN with a [−1,−1,−1 / −1,8,−1 / −1,−1,−1] kernel detects edges. Deep CNNs stack hundreds of such layers — complexity emerges automatically through training.

08 — Developer Takeaways

What This Means for You

  • Prompt engineering — system prompt + history are just more tokens. Context matters.
  • Hallucinations — the model is sampling a distribution. It predicts plausible continuations, not facts.
  • Tool calling — the wrapper routes structured output tokens to real function calls.
  • RAG — inject relevant documents into context before asking. Same mechanism.
  • Fine-tuning — RLHF on a smaller domain-specific dataset.
The magic isn't magic — it's matrix multiplication at scale, trained on human knowledge, guided by human feedback.
